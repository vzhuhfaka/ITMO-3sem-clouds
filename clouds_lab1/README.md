## Для каждой записи определены 5 уровней классификации:

- IT Tower (основная категория: Compute, Database, Storage, Networking и т.д.)
- Service Family (подкатегория внутри IT Tower)
- Service Type (конкретный сервис AWS)
- Service Sub Type (вариант использования сервиса)
- Service Usage Type (детальный тип потребления)

Пример иерархии:<br>
IT Tower: Compute<br>
Service Family: Compute<br>
Service Type: Amazon EC2<br>
Service Sub Type: Dedicated Host<br>
Service Usage Type: Dedicated Host Usage

### Классифицированные сервисы AWS и их аналоги в российских облаках
## Вычислительные сервисы (Compute)
- Amazon EC2 — виртуальные серверы с различными типами инстансов (стандартные, выделенные хосты, GPU-инстансы). Позволяет запускать приложения на виртуальных машинах с разными конфигурациями.
Аналоги в РФ: Яндекс Облако Compute Cloud

- AWS CodeBuild — полностью управляемый CI/CD сервис для сборки и тестирования кода без необходимости управлять инфраструктурой.
Аналоги в РФ: GitLab CI/CD на платформах российских облаков

## Сервисы хранения (Storage)
- AWS Backup — централизованный сервис резервного копирования с поддержкой холодного и теплого хранения, автоматическим управлением жизненным циклом данных.
Аналоги в РФ: VK Cloud Solutions Backup

- Amazon DynamoDB — полностью управляемая NoSQL база данных с моделью оплаты за пропускную способность (Read/Write Capacity Units), автоматическим масштабированием.
Аналоги в РФ: Яндекс YDB (Yandex Database)

## Сетевые сервисы (Networking)
- Amazon EC2 Load Balancer — распределение входящего трафика между несколькими EC2-инстансами для обеспечения отказоустойчивости.
Аналоги в РФ: Яндекс Облако Network Load Balancer

- Amazon EC2 Elastic IP — статические IPv4-адреса для динамического облачного вычисления, позволяющие переназначать адреса между инстансами.
Аналоги в РФ: Плавающие IP-адреса во всех крупных российских облаках

## Управляемые сервисы (Cloud Services)
- Amazon CloudWatch — сервис мониторинга и наблюдения для ресурсов AWS и приложений, работающих на AWS.
Аналоги в РФ: Яндекс Облако Monitoring

- AWS Device Farm — облачный сервис для тестирования мобильных приложений на реальных устройствах и эмуляторах.
Аналоги в РФ: Решения на основе BrowserStack в российских облаках

- Amazon Translate/Transcribe/Comprehend — готовые AI-сервисы для обработки естественного языка: перевод, расшифровка аудио, анализ тональности.
Аналоги в РФ: Яндекс SpeechKit

## Сервисы баз данных (Database)
- Amazon DAX — полностью управляемый кэш в памяти для DynamoDB, ускоряющий время отклика до микросекунд.
Аналоги в РФ: Redis/Memcached решения в российских облаках

- AWS Database Migration Service — сервис для безопасной миграции баз данных в облако AWS или между облачными сервисами AWS.
Аналоги в РФ: Инструменты миграции в Яндекс Облаке
